## URL Base

A API do Mid Poker é construída em princípios REST e servida via HTTPS. Para garantir a privacidade dos dados, HTTP não criptografado não é suportado.

A URL base para todos os endpoints é:

```bash
https://api.mid.poker
```

## Autenticação

A autenticação é feita via header Authorization com um Bearer token:

```bash
Authorization: Bearer <Seu-Token>
```

### Obtendo um Token

Tokens de API podem ser gerados no dashboard em **Configurações > API Keys**.

## Rate Limiting

A API implementa rate limiting para garantir a estabilidade do serviço:

- **1000 requisições** por minuto por token
- Headers de rate limit incluídos nas respostas:
  - `X-RateLimit-Limit`: Limite total
  - `X-RateLimit-Remaining`: Requisições restantes
  - `X-RateLimit-Reset`: Timestamp de reset

## Formato de Resposta

Todas as respostas são retornadas em JSON:

```json
{
  "data": { ... },
  "meta": {
    "total": 100,
    "page": 1,
    "limit": 20
  }
}
```

### Erros

Erros seguem o formato padrão HTTP:

```json
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Token inválido ou expirado"
  }
}
```

## Endpoints Principais

| Endpoint | Descrição |
|----------|-----------|
| `GET /transactions` | Listar transações |
| `GET /accounts` | Listar contas bancárias |
| `GET /accounts/:id/balance` | Saldo de uma conta |
| `GET /institutions/search` | Buscar instituições bancárias |
| `POST /auth/gocardless/link` | Criar link GoCardless |
| `POST /auth/plaid/link` | Criar link Plaid |
| `GET /health` | Status da API |

## SDKs e Bibliotecas

Atualmente, a API pode ser consumida diretamente via HTTP ou através do cliente tRPC incluído no projeto:

```typescript
import { trpc } from '@/trpc/client';

// Exemplo de uso
const transactions = await trpc.transactions.list.query({
  limit: 50,
  offset: 0
});
```
