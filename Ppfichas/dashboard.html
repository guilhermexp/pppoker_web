FASE 1 TESTADO

Envia fichas PPPoker para qualquer nivel da hierarquia do clube (jogador, agente ou super agente). O servidor roteia automaticamente pelo papel do destinatario. Valores internos multiplicados por 100 (centavos).

ParamTipoObrig?DefaultDescricao
target_idintegerSIM—UID de quem vai RECEBER as fichas. Ex: 11470719
amountintegerSIM—Quantidade de fichas (valor face, positivo). Protocolo multiplica x100
clube_idintegerNAO4191918ID numerico do clube
liga_idintegerNAO3357ID da liga/federacao
pb.AddCoinREQ → pb.AddCoinRSP + pb.ClubAgentPPCoinRSP
Payload fields: f1=clube_id, f4=liga_id, f5=0 (type), f6=target_id, f7=amount*100, f8=txn_id (string "{clube}_{sender}_{ts}")
Erros conhecidos:
  • code=0 — Sucesso
  • Saldo insuficiente — retorna erro no AddCoinRSP

HTTP Login — Crypto Chain

POST api.pppoker.club/poker/api/login.php

1. password → MD5 → hex lowercase
2. hex → MD5 → hex lowercase  (= CryptoPassword)
3. CryptoPassword → XXTEA(key) → Base64

Chave XXTEA: MMDDHHMMSS + d5659066d5 (hora Beijing UTC+8)

Exemplo: 02/21 09:28:46 → 0221092846d5659066d5

Probe Pattern — Script de Varredura

1. Login (HTTP + TCP) automatico
2. Entrar no clube alvo (ClubInfoREQ)
3. Testar dezenas de nomes em loop
4. Drenar buffer com timeout 3-5s para cada probe
5. Filtrar broadcasts: {HeartBeatRSP, NoticeBRC, CallGameBRC, PushBRC}
6. Registrar HITS (mensagens que retornaram resposta relevante)

Timeouts recomendados:
  Probing (scan):     3-4s
  info/listar:        5-6s
  export_data:        8s
  transfer_chips:     1-2s
  listar_mesas:       10s (resposta 64KB)
