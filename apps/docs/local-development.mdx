## Introdução

O Mid Poker utiliza um monorepo gerenciado pelo [Turborepo](https://turbo.build/repo) com [Bun](https://bun.sh/) como runtime e package manager.

### Estrutura do Monorepo

```
apps/
├── api          # Backend tRPC + Hono
├── dashboard    # Aplicação Next.js 16
├── docs         # Documentação (Mintlify)

packages/
├── db           # Schema e queries (Drizzle ORM)
├── email        # Templates de email (React Email)
├── invoice      # Geração de faturas PDF
├── jobs         # Background jobs (Trigger.dev)
├── supabase     # Cliente e middleware Supabase
├── ui           # Componentes UI (Shadcn)
├── utils        # Utilitários compartilhados
```

### Apps

- `api`: Backend com tRPC e Hono, servindo em `localhost:8080`
- `dashboard`: Aplicação principal Next.js, servindo em `localhost:9000`
- `docs`: Documentação usando Mintlify

### Packages

- `db`: Schema do banco de dados com Drizzle ORM
- `email`: Templates de email usando React Email
- `invoice`: Geração de faturas em PDF
- `jobs`: Jobs em background com Trigger.dev
- `supabase`: Cliente e utilitários Supabase
- `ui`: Biblioteca de componentes UI
- `utils`: Funções utilitárias compartilhadas

## Executando Localmente

### Pré-requisitos

- [Bun](https://bun.sh/) >= 1.0
- [Node.js](https://nodejs.org/) >= 20
- Uma conta [Supabase](https://supabase.com/)

## Passo 1: Setup Inicial

<Steps>

  <Step title="Clone o repositório">

```bash
git clone https://github.com/seu-usuario/mid.git
cd mid
```

  </Step>

  <Step title="Instale as dependências">

```bash
bun install
```

  </Step>

  <Step title="Configure as variáveis de ambiente">

```bash
cp .env.example .env.local
```

Edite o `.env.local` com suas credenciais do Supabase e outros serviços.

  </Step>

  <Step title="Inicie o servidor de desenvolvimento">

```bash
bun run dev
```

  </Step>

</Steps>

## Passo 2: Configuração do Supabase

1. Crie um novo projeto no [Supabase](https://supabase.com/)
2. Copie a URL e as chaves de API para o `.env.local`
3. Execute as migrations do banco de dados

```bash
cd packages/db
npx drizzle-kit push
```

## Variáveis de Ambiente Necessárias

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://seu-projeto.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=sua-anon-key
SUPABASE_SERVICE_ROLE_KEY=sua-service-role-key
DATABASE_SESSION_POOLER=postgresql://...

# Email (Resend)
RESEND_API_KEY=re_...

# AI (OpenAI)
OPENAI_API_KEY=sk-...

# Opcional
MISTRAL_API_KEY=...
GOOGLE_GENERATIVE_AI_API_KEY=...
```

## Comandos Úteis

```bash
# Desenvolvimento (todos os apps)
bun run dev

# Apenas dashboard
bun run dev --filter=@midpoker/dashboard

# Apenas API
bun run dev --filter=@midpoker/api

# Build de produção
bun run build

# Typecheck
bun run typecheck

# Lint
bun run lint
```
