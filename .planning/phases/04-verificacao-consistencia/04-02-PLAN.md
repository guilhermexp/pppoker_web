<task type="auto">
  <name>Task 3: Auditar RLS policies</name>
  <files>packages/db/migrations/*.sql, packages/db/setup-functions.sql</files>
  <action>
Verificar Row Level Security policies para tabelas poker.

**Conceito de RLS:**
- Supabase usa RLS para isolar dados por team
- Cada query deve filtrar por team_id automaticamente
- Se RLS não configurado, dados de outros teams são acessíveis

**Para cada tabela poker, verificar:**
1. **RLS enabled?**
   - `ALTER TABLE poker_X ENABLE ROW LEVEL SECURITY`

2. **Policies definidas?**
   - SELECT policy (pode ler dados do seu team)
   - INSERT policy (pode inserir dados no seu team)
   - UPDATE policy (pode atualizar dados do seu team)
   - DELETE policy (pode deletar dados do seu team)

3. **Policy conditions:**
   - Usa auth.uid() ou user_id da sessão?
   - Filtra por team_id corretamente?
   - Permite bypass para service role?

**Tabelas a verificar:**
```
poker_players (team_id)
poker_sessions (team_id)
poker_session_players (via session -> team_id)
poker_chip_transactions (team_id)
poker_settlements (team_id)
poker_week_periods (team_id)
poker_imports (team_id)
poker_player_summary (team_id)
poker_player_detailed (team_id)
poker_demonstrativo (team_id)
poker_agent_rakeback (team_id)
```

**Criar tabela de RLS:**
| Table | RLS Enabled | SELECT | INSERT | UPDATE | DELETE | Notes |
|-------|-------------|--------|--------|--------|--------|-------|
  </action>
