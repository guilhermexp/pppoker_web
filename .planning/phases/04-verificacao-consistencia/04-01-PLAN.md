Criar queries de verificação de consistência baseadas nos achados anteriores.

**De 03-03 (chip_balance):**
```sql
-- Verificar se chip_balance = SUM(transactions)
SELECT
  p.id,
  p.chip_balance as stored,
  COALESCE(SUM(
    CASE
      WHEN t.type IN ('chip_in', 'credit_received', 'bonus',
                      'adjustment_positive', 'transfer_in', 'rake_refund', 'jackpot')
      THEN t.amount
      ELSE -t.amount
    END
  ), 0) as calculated
FROM poker_players p
LEFT JOIN poker_chip_transactions t ON t.player_id = p.id
GROUP BY p.id
HAVING p.chip_balance != calculated;
```

**De 03-02 (rake vs settlement):**
```sql
-- Verificar se settlement.grossAmount == SUM(rake) do período
SELECT
  s.id,
  s.gross_amount,
  SUM(ps.rake) as period_rake
FROM poker_settlements s
JOIN poker_player_summary ps ON ps.player_id = s.player_id
WHERE ps.week_period_id = s.week_period_id
GROUP BY s.id
HAVING s.gross_amount != period_rake;
```

**De 02-02 (duplicates):**
```sql
-- Verificar transações duplicadas
SELECT
  player_id,
  week_period_id,
  amount,
  type,
  COUNT(*) as count
FROM poker_chip_transactions
GROUP BY player_id, week_period_id, amount, type
HAVING COUNT(*) > 1;
```

**Criar queries para verificar:**
1. chip_balance consistency
2. Settlement amounts vs rake
3. Duplicate transactions
4. Duplicate demonstrativo records
5. Orphan transactions (player_id = NULL)
6. Settlements para players inexistentes
