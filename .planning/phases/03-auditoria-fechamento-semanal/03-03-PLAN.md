<task type="auto">
  <name>Task 2: Auditar cálculo de chip_balance</name>
  <files>packages/db/src/schema.ts, apps/api/src/trpc/routers/poker/imports.ts</files>
  <action>
Analisar como chip_balance é calculado e mantido.

**Localizar no schema:**
- poker_players.chip_balance - Campo de balanço
- poker_chip_transactions.amount - Valor da transação
- poker_chip_transactions.type - Tipo da transação

**Verificar a fórmula:**
```
chip_balance = SUM(transações positivas) - SUM(transações negativas)

ou

chip_balance = SUM(transactions WHERE impacto = '+')
             - SUM(transactions WHERE impacto = '-')
```

**Questões críticas:**
1. chip_balance é calculado em runtime ou armazenado?
2. Se armazenado, quando é atualizado?
3. Há trigger no banco ou é calculado no código?
4. O que acontece quando transação é deletada?
5. chip_balance é resetado após closeWeek?

**Verificar consistência:**
```
SELECT player_id,
       chip_balance as stored,
       SUM(CASE WHEN type IN (...) THEN amount ELSE -amount END) as calculated
FROM poker_players p
JOIN poker_chip_transactions t ON t.player_id = p.id
GROUP BY player_id
HAVING stored != calculated
```

**Edge cases:**
- Player sem transações (balance = 0 ou NULL?)
- Transação com amount = 0
- Transação com player_id = NULL (de 02-02: é permitido)
- Re-importação de dados (duplica transações)
  </action>
  <verify>Fórmula de chip_balance documentada e verificada</verify>
  <done>Cálculo de chip_balance auditado</done>
</task>
