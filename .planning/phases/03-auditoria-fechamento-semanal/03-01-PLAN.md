<task type="auto">
  <name>Task 2: Auditar closeWeek logic em week-periods</name>
  <files>apps/api/src/trpc/routers/poker/week-periods.ts, apps/api/src/schemas/poker/week-periods.ts</files>
  <action>
Ler e analisar o procedure `closeWeek` (~1,194 linhas no router).

O `closeWeek` é o procedure mais crítico - gera settlements automaticamente.

**Documentar:**
1. **Trigger:** Como é chamado (frontend CloseWeekPreviewModal)
2. **Precondições:** Validações antes de fechar
3. **Lógica de geração de settlements:**
   - Quais players são incluídos?
   - Como chip_balance é calculado?
   - Como rakeback é aplicado?
   - Como agent settlements são gerados?
4. **Atomicidade:** É uma transação atômica?
5. **Rollback:** O que acontece se falhar no meio?
6. **Status tracking:** week_period status (open → closed → locked)

**Verificar matematicamente:**
```
Para cada player com chip_balance != 0:
  grossAmount = chip_balance
  rakebackAmount = grossAmount * rakeback_percent / 100
  netAmount = grossAmount - rakebackAmount

  INSERIR settlement com esses valores
  RESETAR chip_balance para 0
```

**Edge cases críticos:**
- Player sem rakeback_percent definido
- Player com chip_balance = 0
- Agente com múltiplos players
- Re-fechamento de semana já fechada
- Dados importados após close
  </action>
  <verify>closeWeek logic documentado com todos os edge cases</verify>
  <done>closeWeek auditado com fluxo completo e edge cases mapeados</done>
</task>
